# 产品路线图

## 阶段 1：首版业务闭环

目标是提供可以实际使用和验收的核心链路。

| 工作包 | 交付内容 | 状态 |
| --- | --- | --- |
| 基础平台 | 认证、用户、角色、权限、动态功能导航、管理后台框架 | 待重写 |
| 模型能力 | 模型配置、默认模型、测试调用、调用记录 | 待重写 |
| 智能瞭望 | 数据源、规则、采集任务、执行结果 | 待重写 |
| 数据沉淀 | 内容存储、去重、筛选与可用状态治理 | 待重写 |
| 智能问数 | 会话、问题、检索依据、流式回答与引用 | 待重写 |
| 基础安全 | 敏感配置保护、采集边界、权限检查、审计 | 待重写 |

## 阶段 2：使用体验与运营能力

在首版真实使用反馈成立后评估：

- 数据统计和基础运营视图。
- 采集任务调度、失败重试和通知能力。
- 更丰富的来源解析适配能力。
- 问答质量评估、反馈与知识内容维护流程。

## 团队任务书扩展交付

2026-05-31 起，课程团队任务书已明确要求交付即时通讯、数字员工、舆情大屏、视觉/语音增强、自动化和多数据库能力。以上内容不再只是候选方向，但仍位于首版 MVP 之后，按 `docs/planning/course-assignment-delivery-plan.md` 的 Phase 5-10 推进。

## 其他候选扩展池

以下能力仍属于 MVP 和课程任务书以外的候选方向，不是既定承诺：

| 方向 | 候选能力 | 启动前提 |
| --- | --- | --- |
| 即时通讯增强 | 消息撤回、消息搜索、音视频通话、复杂群权限 | 课程要求的聊天主链路已稳定 |
| 数字员工增强 | 长期记忆、复杂工具编排、外部插件市场 | 课程要求的角色化助手和工具绑定已稳定 |
| 舆情增强 | 复杂报告模板、推荐算法、实时告警策略 | 基础统计和分析结果已稳定 |
| 基础设施增强 | 微服务拆分、跨地域容灾、独立消息中间件 | 真实规模、部署或团队边界提出要求 |

## 变更规则

- 将候选能力升级为正式范围，必须记录到 `docs/context/decisions.md`。
- 新阶段开始前，需更新 `docs/context/current-status.md` 和相关数据库/API 契约。
