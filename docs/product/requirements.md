# 首版正式需求

## 1. 范围与角色

首版交付智能瞭望与智能问数主链路，并提供支撑该链路的管理后台。

| 角色 | 能力范围 |
| --- | --- |
| 系统管理员 | 用户、角色、权限、模型、数据源、采集任务、数据内容和审计管理 |
| 业务管理员 | 经授权后维护模型、数据源、采集任务和数据内容 |
| 普通用户 | 经授权后使用智能问数并查看自己的问答记录 |

## 2. 认证、基础 RBAC 与功能导航

- 系统提供登录、退出和当前用户会话识别。
- 管理员可以创建、停用和维护用户，不可读取任何用户明文密码。
- 管理员可以维护角色，并将权限分配给角色、将角色分配给用户。
- 管理员可以维护后台功能导航的展示名称、层级、排序、图标、目标页面和启用状态。
- 功能导航项可以关联访问所需的权限代码；无权限的用户不展示对应入口。
- 后台页面和后台 API 均必须执行服务端权限检查。
- 内置超级管理员角色与首个系统管理员账户不可被普通管理操作删除或失去管理权限。
- 功能导航仅负责入口配置和展示控制，不得替代后端权限判断，也不得让不存在的业务模块因配置菜单而自动可用。

**验收结果**：管理员可调整后台菜单展示；不同权限账号所见入口和服务端可执行操作均符合授权范围。

## 3. 模型引擎

- 管理员可以新增、编辑、启用或停用兼容 OpenAI Chat Completions 范式的模型配置。
- 系统允许指定唯一的默认启用模型，问数流程默认使用该模型。
- 管理员可以发起模型连接/对话测试，并选择普通响应或 SSE 流式响应。
- 系统记录模型调用用途、成功状态、耗时和可获得的 token 使用量。
- API 密钥等认证信息必须加密保存，不得通过查询 API 回传明文。

**验收结果**：管理员可以配置一个有效模型并完成流式测试；系统能够在不泄露密钥的情况下展示调用统计。

## 4. 智能瞭望

- 管理员可以维护采集数据源，包括名称、入口地址、允许访问范围、启用状态和说明。
- 管理员可以为数据源维护规则，包括请求方式、非敏感请求参数、内容解析规则和结果映射配置。
- 系统支持针对一个或多个启用数据源创建采集任务。
- 采集任务可查看执行状态、开始结束时间、成功/失败数量和失败原因摘要。
- 采集发起前必须执行地址与网络安全校验；不得允许规则执行任意服务端脚本。

**验收结果**：管理员可配置来源、运行任务并看到成功入库或可诊断的失败结果。

## 5. 数据仓库

- 系统保存采集获得的内容及来源、采集时间、原始链接、摘要、状态和去重标识。
- 管理员可以按来源、关键词、时间和状态筛选数据内容。
- 管理员可以将内容标记为可用、归档或排除，以控制其是否可用于问数。
- 重复内容不得重复进入可用知识集合；应保留必要的采集追踪信息。

**验收结果**：经过治理的有效内容能够被问数流程选取；排除或归档内容不会作为正常依据使用。

## 6. 智能问数

- 授权用户可以创建问数会话、提交问题并查看自己的历史会话。
- 系统根据问题检索可用数据内容，将相关依据与问题组合后调用默认模型。
- 回答支持 SSE 流式输出。
- 回答完成后，系统保存问题、回答、模型调用信息及被引用的数据内容。
- 前端展示回答及引用依据，用户能够查看依据标题、来源和内容摘要。

**验收结果**：用户完成一次问题提交，获得流式回答，并能够识别回答引用的系统数据。

## 7. 安全与操作记录

- 密码采用可靠的慢哈希算法存储；会话和 CSRF 防护必须启用。
- 模型密钥、采集认证头或 Cookie 等敏感配置加密保存并脱敏展示。
- 采集模块必须防范 SSRF，不得访问本机、内网、元数据服务或不在允许范围内的目标。
- 高风险操作记录操作者、动作、目标、结果和时间，包括权限变更、模型密钥更新、数据源更新、采集任务发起和内容状态调整。

## 8. 首版不包含

- 即时通讯、群聊或文件消息。
- 数字员工与外部工具编排。
- 舆情分析、可视化大屏、语音或手势交互。
- 定时工作流、异常告警和多数据库动态切换。
