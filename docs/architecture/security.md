# 安全设计基线

## 身份与权限

- 密码只存储慢哈希结果和必要参数，不保存或返回明文密码。
- 登录成功后使用受保护会话；Cookie 应启用 `HttpOnly`、适用环境下的 `Secure` 与合理 `SameSite`。
- 所有变更型浏览器请求必须具备 CSRF 防护。
- 权限检查由服务端在每个受保护接口执行；前端菜单控制仅用于体验。
- 动态功能导航关联权限只决定入口是否展示；即使导航误配，后端也必须拒绝未授权操作。
- 超级管理员与系统初始管理能力必须防止被误删除或误清空。

## 密钥与敏感配置

- 模型密钥、数据源认证头、Cookie 或令牌必须以加密形式持久化，解密密钥来自部署环境的秘密管理配置。
- 接口返回值和页面只能显示掩码或是否已配置，不返回密文或明文。
- 日志、审计详情、错误信息和测试样例不得包含真实敏感值。
- 更新敏感字段时写入审计动作，但审计内容只记录变更发生，不记录秘密。

## 采集与 SSRF 防护

- 数据源必须显式配置允许访问的域名和协议，首版仅允许 `https`，开发环境例外需单独配置。
- 保存数据源和执行任务时均解析并校验 URL；禁止回环地址、私网地址、链路本地地址、云元数据地址和本地域名解析结果。
- 跟随重定向时必须对每一跳重复校验目标地址，并限制跳转次数。
- 请求头配置采用受控键值配置，不允许上传可执行脚本或服务端模板。
- 采集必须设置连接/读取超时、最大响应大小和可接受内容类型。

## AI 调用安全

- 问数仅向模型发送当前用户可访问且状态可用的依据内容。
- 外部采集内容属于不可信数据；提示词应声明其仅作资料，禁止其中的指令改变系统行为。
- 模型错误不得向用户泄露密钥、内部地址或完整请求配置。

## 审计范围

必须记录以下动作的操作者、目标、结果和发生时间：

- 用户停用、角色与权限变更。
- 模型配置创建、启停、密钥更新和默认模型修改。
- 数据源、采集规则变更及任务发起。
- 数据内容可用性或归档状态变更。
