# 可复用开发 SOP

## Skill：新增 MVC 业务模块

1. 阅读正式需求、架构模块边界及当前状态。
2. 明确该模块负责与不负责的行为，确认所需权限代码。
3. 若产生新实体或接口，先更新数据库/API 契约。
4. 在 Controller、Model、View 中实现最小完整业务链。
5. 补充权限、校验、失败路径与审计。
6. 验证核心流程并更新动态上下文文档。

## Skill：修改数据库契约并实施迁移

1. 在 `docs/database/schema.md` 和 `data-rules.md` 描述业务原因与新规则。
2. 检查对 API、权限、审计和历史数据的影响。
3. 编写可重复执行或受版本管理的迁移，不依赖人工修改生产数据。
4. 更新 Model 逻辑与测试数据构造。
5. 验证迁移前后关键行为和数据约束。
6. 将决定和风险记录到上下文文档。

## Skill：新增或调整 API

1. 确认调用角色、权限代码、资源归属和敏感边界。
2. 更新对应 `docs/api/*.md`，定义路径、输入、响应和失败状态。
3. 在 Controller 实现校验、授权和统一响应。
4. 在 Model 实现事务、状态规则及必要审计。
5. 验证成功、校验失败、未授权和业务冲突场景。

## Skill：接入模型供应商

1. 确认供应商符合 OpenAI 兼容调用边界，或先扩展正式协议设计。
2. 将凭据作为秘密处理，只保存加密值和掩码。
3. 实现连接测试、启停和默认模型规则。
4. 对普通与 SSE 调用统一记录脱敏调用统计。
5. 验证上游失败、超时、无默认模型和密钥不可见场景。

## Skill：新增采集源或采集流程

1. 明确数据源范围、数据用途、允许域名和解析需求。
2. 为来源与规则设计受校验配置，不接受执行脚本。
3. 落实 SSRF 校验、超时、大小限制、重定向复核和凭据保护。
4. 将标准化结果写入数据仓库并按契约去重。
5. 验证成功采集、不可访问目标、解析失败与部分失败任务。

## Skill：新增智能问数能力

1. 确认用户权限、会话归属和允许检索的数据范围。
2. 只选取可用内容作为依据，保存实际引用摘录。
3. 通过模型引擎调用默认启用模型，并使用 SSE 返回片段。
4. 将外部内容作为不可信资料隔离，避免提示注入改变系统行为。
5. 验证有依据回答、无依据回答、模型失败、中断流和历史引用访问。

## Skill：更新项目上下文

1. 在 `current-status.md` 更新已验收能力而非仅已编码能力。
2. 在 `decisions.md` 写入后续实现必须遵守的新选择。
3. 在 `issues.md` 写入尚未解决的风险和阻塞。
4. 在 `ai-memory.md` 留下短小、可复用且不含秘密的已确认事实。

## Skill：交付变更供 Code Review

1. 阅读 `docs/workflow/git.md` 并检查当前 Git 分支与工作树。
2. 确保变更位于工作分支，不在主分支直接提交。
3. 运行相关验证，检查未纳入秘密、本地配置或无关文件。
4. 使用 Conventional Commits 提交可独立审查的变更。
5. 向项目负责人提供变更摘要、验证结果、风险和契约变更说明。
6. 根据评审意见继续在同一分支修改；未经明确审核通过，不合并主分支。
